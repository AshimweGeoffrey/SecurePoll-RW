Offline Sync Architecture

~4 hours offline operation
per polling station
double-vote safe without network
1

Pre-load

Before election day
Redis biometric pre-cache
Templates for the station's roster pushed from the server
Encrypted bundle → tablet
Written into local SQLite, AES-encrypted at rest
Integrity check
Station confirms roster & cache before polls open
2

Offline operation

Network dropped
Local verification
Face / fingerprint matched on-device against the cache
Bloom filter = authority
Local "voted set" blocks double-voting until sync
Local SQLite (encrypted)
Biometric cache pre-loaded
Verification queue pending sync
Voted bloom filter double-vote lock
3

Reconnect & reconcile

Network restored
POST /verify/sync
Batch of queued verifications sent to Verification Service
Server processes queue
Sequential apply · row-locked VOTED writes to Voter DB
sync.completed
Election Ops + Notification updated; turnout reconciled
push cache
on reconnect
End-of-day reconciliation flags any station where the physical ballot count ≠ the system count — the offline bloom filter guarantees a voter can never be checked in twice, even with zero connectivity.
